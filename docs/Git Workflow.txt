

GIT REBASE WORKFLOW

1. git checkout master
2. git pull --rebase upstream master
3. git checkout -b feature-branch
4. git add/git commit (on feature branc)
5. git pull --rebase upstream master
6. git push origin feature-branch
7. Submit pull request (your feature branch to origin master)

MERGE?
Yes
1. git checkout master
2. git pull --rebase upstream master

No
1. make more commits to feature-branch
2. git push origin feature-branch




Troubleshooting
* git remote -v to see remote origins
* git remote add upstream https://github.com/mathematicalthinking/encompass.git
* if existing upsteam git remote rm upstream
